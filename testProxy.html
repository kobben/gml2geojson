gml2geojson proxy examples

Testing gml2geojson proxy (http://kartoweb.itc.nl/gml2geojson/gml2geojson.py?url=)

Most basic format url to provide: base url + TYPENAME
http://geodata.nationaalgeoregister.nl/bestuurlijkegrenzen/wfs?TYPENAME=bestuurlijkegrenzen:provincies

TYPENAME is required for ogr2ogr to request layer
http://geodata.nationaalgeoregister.nl/bestuurlijkegrenzen/wfs?

TYPENAMES (WFS 2.0.0 spec) is also accepted (but will fail if >1 layer)
http://geodata.nationaalgeoregister.nl/bestuurlijkegrenzen/wfs?TYPENAMES=bestuurlijkegrenzen:provincies

Mapserver MAP parameters are properly attached to base url
http://kartoweb.itc.nl/cgi-bin/mapserv.exe?map=D:/Inetpub/mapserver/wfs_config.map &TYPENAME=AirportsWFS

Limited testing of additional parameters is done (SERVICE, REQUEST, RESULTTYPE)
http://geodata.nationaalgeoregister.nl/wijkenbuurten2016/wfs?&SERVICE=WMS&VERSION=1.1.0&REQUEST=DescribeFeatureType&SRS=EPSG%3A28992&OUTPUTFORMAT=GML2 &TYPENAME=wijkenbuurten2016%3Agemeenten2016&RESULTTYPE=hits

All other parameters are unfiltered: send forward to ogr2ogr, but not garantueed to work. E.g. this does work...
http://kartoweb.itc.nl/gml2geojson/gml2geojson.py?url=http://geodata.nationaalgeoregister.nl/bestuurlijkegrenzen/wfs?SERVICE=WFS&VERSION=1.1.0&REQUEST=GetFeature&SRSNAME=EPSG%3A28992&OUTPUTFORMAT=GML3 &TYPENAME=bestuurlijkegrenzen:provincies

...but this not (because ogr2ogr chokes on the OUTPUTFORMAT)
http://kartoweb.itc.nl/gml2geojson/gml2geojson.py?url=http://geodata.nationaalgeoregister.nl/bestuurlijkegrenzen/wfs?SERVICE=WFS&VERSION=1.1.0&REQUEST=GetFeature&SRSNAME=EPSG%3A28992&OUTPUTFORMAT=text/xml; subtype=gml/3.2 &TYPENAME=bestuurlijkegrenzen:provincies
